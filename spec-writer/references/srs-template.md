## 작성 가이드

- **"~해야 한다" 형식**: 요구사항은 "시스템은 ~해야 한다(shall)" 형식으로 작성.
- **검증 가능성**: 모든 요구사항에 검증 방법을 명시. 검증할 수 없으면 요구사항이 아니라 희망사항.
- **기능/비기능 분리**: 기능은 "무엇을 하는가", 비기능은 "얼마나 잘 하는가".
- **우선순위 밸런스**: P0이 전체의 40%를 넘지 않도록 조절. 모든 것이 긴급하면 아무것도 긴급하지 않다.
- **비기능 요구사항에도 수치를 붙인다**: "빠르게" ✗ → "200ms 이내" ✓
