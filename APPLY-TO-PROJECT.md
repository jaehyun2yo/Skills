# 다른 프로젝트에 스킬 적용하기

## 사전 준비 (1회만)

```bash
# Skills 레포 클론 (이미 했으면 스킵)
git clone https://github.com/jaehyun2yo/Skills.git ~/Skills

# 전체 스킬 글로벌 심링크
mkdir -p ~/.claude/skills
for skill in ~/Skills/*/; do
  name=$(basename "$skill")
  [ -f "$skill/SKILL.md" ] && ln -sf "$skill" ~/.claude/skills/"$name"
done
```

---

## 적용 프롬프트 (대상 프로젝트에서 실행)

아래 프롬프트를 대상 프로젝트 디렉토리에서 Claude Code에 붙여넣기:

---

```
이 프로젝트에 개발 환경을 셋업해줘. 아래 순서대로 진행:

1단계: 프로젝트 분석 & 셋업 (project-setup)
- 현재 프로젝트를 스캔하고 48점 기준으로 채점해줘
- 부족한 항목마다 추천 설정을 보여주고, 내가 선택하면 생성해줘
- CLAUDE.md, hooks (settings.json), rules, 운영문서 전부 포함

2단계: 스킬 등록 (skill-register)
- 글로벌 스킬이 이 프로젝트의 CLAUDE.md에 등록되었는지 확인
- 안 되어있으면 Skill Triggers 테이블에 등록해줘

3단계: 팀 셋업 (team-setup)
- 프로젝트 구조를 분석해서 도메인별 에이전트 팀 스킬을 생성해줘
- 팀 디스패처도 같이 설정

4단계: 사용법 생성 (usage-guide)
- 위에서 설정한 모든 것을 포함한 통합 사용 가이드를 docs/usage-guide.md로 생성

5단계: 최종 검증
- project-setup으로 다시 채점해서 점수 변화를 보여줘
- 빠진 항목이 있으면 알려줘

각 단계 완료 후 결과를 보여주고 다음 단계로 넘어가줘.
```

---

## 간단 버전 (한 줄)

```
셋업해줘
```

project-setup 스킬이 자동 트리거되어 전체 프로세스를 안내합니다.

## 단계별 적용 (선택 적용)

특정 기능만 필요하면 개별 트리거:

| 목적 | 프롬프트 |
|------|---------|
| 프로젝트 점검만 | `프로젝트 설정 점검해줘` |
| 스킬 등록만 | `스킬 등록해줘` |
| 팀 구성만 | `팀 셋업해줘` |
| 사용법 문서만 | `사용법 만들어줘` |
| 보안 검사 | `보안 검사해줘` |
| 코드 정리 | `코드정리해줘` |
| 스킬 품질 측정 | `스킬 평가해줘` |
| E2E QA 테스트 | `QA 테스트해줘` |

## 적용 후 일상 워크플로우

```
세션 시작: "시작하자"     → 핸드오프 로드 + 상태 표시 + 다음 작업 제안
코딩 중:   (자동)         → PostToolUse 포맷팅, PreToolUse 보안 차단
컨텍스트:  /clear         → 자동 핸드오프 저장 → 새 세션에서 자동 복원
세션 종료: "마무리"       → 코드정리 → 커밋 → 진행상황 기록 → 핸드오프 생성
```
