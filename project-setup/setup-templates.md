### Session Protocol Section
```markdown
## Session Protocol

**Start:**
1. Read `docs/progress.txt` — what happened last session
2. Read `docs/features-list.md` — identify next task
3. Read relevant spec (if exists) before coding
4. Plan before code — never start without a plan

**End:**
1. Commit all changes
2. Update `docs/progress.txt`
3. Update `docs/features-list.md` status
4. Update `docs/changelog/CHANGELOG.md`
5. Update specs if code changed behavior

**Stuck (2+ failed fix attempts):**
1. STOP — do not attempt a third time
2. Commit current state
3. Record in progress.txt: what tried, why failed, which files
4. /clear → new session → different approach
```

### Context Management Section
```markdown
## Context Management

- 0-50%: work freely
- 50-70%: use Grep, avoid full file reads
- 70-80%: /compact (manual, before auto-compression is more effective)
- 90%+: /clear (handoff.md is auto-generated by PreCompact hook)
- Never read files > 500 lines in full — use Grep for targeted search
{{LARGE_FILE_WARNINGS}}

## Session Handoff (Automatic)
- PreCompact hook auto-saves state to `.claude/handoff.md` before compaction
- SessionStart hook auto-loads `.claude/handoff.md` after /clear or compaction
- No manual action needed — just /clear when context is full
- Handoff includes: completed work, pending tasks, decisions, failed approaches (Lore), next steps
```

### settings.json (with hooks + auto session handoff)
```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "startup",
        "hooks": [
          {
            "type": "command",
            "command": "echo '=== Git ===' && git branch --show-current 2>/dev/null && git status --short 2>/dev/null && echo '' && echo '=== Recent Commits ===' && git log --oneline -5 2>/dev/null && echo '' && echo '=== Progress ===' && head -25 docs/progress.txt 2>/dev/null || echo 'no progress.txt' && echo '' && echo '=== Pending ===' && grep -c 'FAILING\\|❌\\|⚠️' docs/features-list.md 2>/dev/null || echo 'no features-list.md'"
          }
        ]
      },
      {
        "matcher": "compact",
        "hooks": [
          {
            "type": "command",
            "command": "echo '=== Session Handoff (auto-loaded after compaction) ===' && cat .claude/handoff.md 2>/dev/null || echo 'No handoff file found'"
          }
        ]
      },
      {
        "matcher": "clear",
        "hooks": [
          {
            "type": "command",
            "command": "echo '=== Session Handoff (auto-loaded after /clear) ===' && cat .claude/handoff.md 2>/dev/null || echo 'No handoff file found' && echo '' && echo '=== Git ===' && git branch --show-current 2>/dev/null && git status --short 2>/dev/null"
          }
        ]
      }
    ],
    "PreCompact": [
      {
        "hooks": [
          {
            "type": "prompt",
            "prompt": "Context is about to be compacted. Before compaction, write a handoff file to .claude/handoff.md with: 1) What was completed this session, 2) What is still in progress (with specific file paths and line numbers), 3) Key decisions made and why, 4) Failed approaches and why they failed (Lore), 5) Next steps in priority order, 6) If Agent Team was used: team composition, task status, and re-spawn prompts. This file will be auto-loaded in the next session."
          }
        ]
      }
    ],
    "Stop": [
      {
        "hooks": [
          {
            "type": "prompt",
            "prompt": "Before finishing, verify: 1) All changes committed? 2) .claude/handoff.md written with session state? 3) docs/progress.txt updated? 4) docs/features-list.md status accurate? 5) Active Agent Team teammates shut down? If all done, respond 'complete'. Otherwise, handle remaining items."
          }
        ]
      }
    ]
  },
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

### How the Auto-Handoff System Works

```
[Coding...]
  ↓ Context reaches compaction threshold
  ↓ PreCompact hook fires → prompt: "Write .claude/handoff.md"
  ↓ Claude writes handoff with: completed, pending, decisions, lore, next steps
  ↓ Compaction or /clear happens
  ↓ SessionStart hook fires (matcher: "compact" or "clear")
  ↓ Hook outputs handoff.md content → injected into new session context
  ↓ Seamless continuation — no manual intervention needed
```

User only needs to type `/clear` — everything else is automatic.

### Alternative: Shell-script based hooks (for complex logic)

session-start.sh:
```bash
#!/bin/bash
# Auto-load handoff if exists (after /clear or compaction)
HANDOFF=".claude/handoff.md"
if [ -f "$HANDOFF" ]; then
  echo "=== Session Handoff Found ==="
  cat "$HANDOFF"
  echo ""
fi

PROGRESS_FILE="docs/progress.txt"
if [ -f "$PROGRESS_FILE" ]; then
  echo "=== Progress ==="
  head -25 "$PROGRESS_FILE"
  echo ""
fi

echo "=== Git ==="
git branch --show-current 2>/dev/null
git status --short 2>/dev/null
echo ""
echo "=== Recent Commits ==="
git log --oneline -5 2>/dev/null
echo ""
echo "=== Pending ==="
grep -c 'FAILING\|❌' docs/features-list.md 2>/dev/null && echo " items remaining" || echo "no features-list.md"
```

session-stop.sh:
```bash
#!/bin/bash
echo "=== Session End Check ==="

# Check handoff file
HANDOFF=".claude/handoff.md"
if [ -f "$HANDOFF" ]; then
  FILE_TIME=$(stat -c %Y "$HANDOFF" 2>/dev/null || stat -f %m "$HANDOFF" 2>/dev/null)
  NOW=$(date +%s)
  DIFF=$((NOW - FILE_TIME))
  if [ "$DIFF" -gt 1800 ]; then
    echo "[!] handoff.md not updated in 30+ minutes."
  else
    echo "[OK] handoff.md recently updated."
  fi
else
  echo "[!] No handoff.md — session state may be lost on /clear."
fi

# Check uncommitted changes
CHANGES=$(git status --porcelain 2>/dev/null | wc -l)
if [ "$CHANGES" -gt 0 ]; then
  echo "[!] Uncommitted changes: ${CHANGES} files"
  git status --short 2>/dev/null
else
  echo "[OK] All changes committed."
fi
```

### CLAUDE.md — Agent Team Section
Add this section to CLAUDE.md when Agent Team is enabled:
```markdown
## Agent Team Guidelines

### Team Model Routing
- **Lead**: Opus (coordination, judgment, synthesis)
- **Implementation teammates**: Sonnet (cost-effective, strong coding)
- **Simple lookup teammates**: Haiku (lowest cost)

### Directory Ownership Map
Assign directories to prevent file conflicts during team work:
| Area | Owner | Directories |
|------|-------|-------------|
{{DIRECTORY_OWNERSHIP_MAP}}

### Team Composition Patterns

**Parallel Implementation** (feature dev, refactoring):
- Lead (Opus): coordinate only, delegate mode
- Teammate per layer (Sonnet): each owns exclusive directories
- Test teammate (Sonnet): tests/ only

**Multi-Perspective Review** (code review, security audit):
- Lead (Opus): synthesize findings
- Security reviewer (Sonnet): auth, injection, data validation
- Performance reviewer (Sonnet): queries, memory, latency
- Test coverage reviewer (Sonnet): edge cases, missing tests

**Adversarial Debug** (complex bugs, investigations):
- Lead (Opus): moderate debate, synthesize consensus
- N hypothesis investigators (Sonnet): each argues different root cause
- Instruction: "Challenge each other's theories like a scientific debate"

**Architecture Decision** (tech choices, design):
- Lead (Opus): produce final ADR document
- N advocates (Sonnet): each argues for different approach
- Instruction: "Challenge each other on performance, cost, complexity, migration effort"

### Team Spawn Best Practices
Always include in spawn prompts:
1. `"Use delegate mode — only coordinate, don't implement yourself"`
2. `"ONLY touch files in [assigned directories]"` — per teammate
3. `"Require plan approval before making any changes"`
4. `"Use Sonnet for each teammate"`
5. Do NOT duplicate CLAUDE.md content in spawn prompts (auto-loaded)

### Team Session Protocol
- `/compact` at 70-80% context (before 95% auto-compression)
- Shut down idle teammates immediately (they consume tokens)
- Always: shutdown all teammates → then clean up team (never reverse)
- In-process teammates cannot be resumed after `/resume` — re-spawn needed

### Team Cost Awareness
- 3 teammates ≈ 3-4x token cost vs single session
- 5 teammates ≈ 7x token cost
- Optimize: Sonnet teammates, concise spawn prompts, prompt cleanup
```
